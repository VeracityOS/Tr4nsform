01

Protocol Overview

What is LNURL-Auth?

LNURL-Auth is a Lightning Network authentication protocol where the user's wallet is their identity. The user never provides an email, phone number, or password. Their Lightning node deterministically derives a unique keypair per domain — meaning the same wallet generates a different identity on every site, making cross-site tracking impossible by design.

BTCPay Server includes native LNURL-Auth support. When combined with a lightweight middleware layer, it becomes the backbone of a fully sovereign user authentication system for any web frontend including Webflow.

⚡
Sovereign Design Principle The Lightning keypair is derived client-side in the wallet. The server never sees a private key, password, or PII. The identity proof is a cryptographic signature over a random challenge — indistinguishable from noise to any observer.

Authentication Flow

1
Frontend Requests Challenge
Webflow JS calls GET /api/lnauth/init on your middleware. The server generates a random 32-byte nonce (k1), encodes it as a LNURL-auth string (bech32), and returns a QR code as a data URI.
2
QR Rendered to User
Webflow displays the QR code. The user scans it with any LNURL-Auth compatible Lightning wallet (Phoenix, Breez, Zeus, Alby, etc.).
3
Wallet Derives Domain-Scoped Keypair
The wallet derives a unique keypair for your domain from the user's Lightning seed. It signs the k1 challenge with the derived private key and POSTs sig + key to your callback URL.
4
Middleware Verifies Signature
The middleware verifies the secp256k1 ECDSA signature against the public key and k1. On success, it marks the challenge as used, upserts the user's profile record keyed by lnkey, and stores the session.
5
Frontend Polls for JWT
Webflow JS polls GET /api/lnauth/verify?k1=... every 2 seconds. Once the wallet callback completes, the endpoint returns a signed JWT. Webflow stores it in localStorage.
6
User Authenticated — Wallet Registration Available
With the JWT, the user can optionally register additional wallet data (on-chain Bitcoin address, alias) via POST /api/profile/register. All fields are optional and pseudonymous.

Architecture & Stack

Auth Protocol
LNURL-Auth
Lightning wallet signs a server challenge. Domain-scoped keypair derived from seed. No PII transmitted.
Auth Server
BTCPay Server
Native LNURL-Auth support at 95.111.241.194. Used for reference or direct routing.
Middleware
Node.js / Express
Custom challenge generation, secp256k1 signature verification, JWT issuance, profile management.
Frontend
Webflow + Custom Code
QR display, wallet link, JWT storage in localStorage, profile UI, registration form.
Session Layer
JWT (7-day)
Signed with HS256. Payload contains lnkey only. Stored client-side.
Profile Store
SQLite or Redis
Three tables: challenges, sessions, profiles. Primary key is the Lightning domain public key.

BTCPay Server Setup

Enable LNURL-Auth

Your BTCPay instance at 95.111.241.194 includes native LNURL-Auth. Navigate to:

Server Settings → Services → Lightning → LNURL-Auth → Enable

Once enabled, the LNURL-Auth callback is available at:

ENDPOINT BTCPay LNURL-Auth callback 
https://95.111.241.194/lnauth/login
⚠
Use Middleware for Custom Profiles BTCPay's native LNURL-Auth handles authentication only — it does not manage custom profile records, BTC address binding, or alias registration. You must deploy the middleware layer in Section 04 to support wallet profile features. BTCPay can be used as a reference point or for Lightning payment integration.
ℹ
MYRA OS Integration Path For unified Lightning identity across the full sovereign stack, route the LNURL-Auth callback through your LNbits instance running on the MYRA OS Docker stack. This allows the same lnkey to authenticate across BTCPay, BUMP, the SMILE Ledger, and any other SASCOSM service surface.

JAVASCRIPT server.js 
cat > server.js << 'EOF'
require("dotenv").config();
const express    = require("express");
const cors       = require("cors");
const jwt        = require("jsonwebtoken");
const Database   = require("better-sqlite3");
const { bech32 } = require("bech32");
const secp256k1  = require("secp256k1");
const QRCode     = require("qrcode");
const crypto     = require("crypto");

const app        = express();
const db         = new Database("profiles.db");
const JWT_SECRET = process.env.JWT_SECRET
                    || crypto.randomBytes(32).toString("hex");
const BASE_URL   = process.env.BASE_URL
                    || "https://auth.yourdomain.com";
const PORT       = process.env.PORT || 3000;

// ── CORS: allow Webflow origins ──────────────────────────────
app.use(cors({ origin: [
  "https://your-webflow-site.webflow.io",
  "https://yourdomain.com"
]}));
app.use(express.json());

// ── DATABASE INIT ─────────────────────────────────────────────
db.exec(`
  CREATE TABLE IF NOT EXISTS challenges (
    k1         TEXT    PRIMARY KEY,
    created_at INTEGER,
    used       INTEGER DEFAULT 0
  );
  CREATE TABLE IF NOT EXISTS profiles (
    lnkey       TEXT PRIMARY KEY,
    btc_address TEXT,
    alias       TEXT,
    metadata    TEXT,
    created_at  INTEGER,
    last_seen   INTEGER
  );
  CREATE TABLE IF NOT EXISTS sessions (
    k1        TEXT PRIMARY KEY,
    lnkey     TEXT,
    authed_at INTEGER
  );
`);

// ── ROUTE: generate challenge ─────────────────────────────────
app.get("/api/lnauth/init", async (req, res) => {
  const k1       = crypto.randomBytes(32).toString("hex");
  const callback = `${BASE_URL}/api/lnauth/callback`;

  db.prepare("INSERT INTO challenges (k1, created_at) VALUES (?, ?)")
    .run(k1, Date.now());

  const lnurlAuthUrl = `${callback}?tag=login&k1=${k1}`;
  const encoded      = bech32.encode(
    "lnurl",
    bech32.toWords(Buffer.from(lnurlAuthUrl)),
    1500
  );
  const qr           = await QRCode.toDataURL(encoded.toUpperCase());

  res.json({ k1, lnurl: encoded.toUpperCase(), qr });
});

// ── ROUTE: wallet callback ────────────────────────────────────
app.get("/api/lnauth/callback", (req, res) => {
  const { k1, sig, key } = req.query;
  if (!k1 || !sig || !key)
    return res.json({ status: "ERROR", reason: "Missing params" });

  const challenge = db
    .prepare("SELECT * FROM challenges WHERE k1 = ? AND used = 0")
    .get(k1);
  if (!challenge)
    return res.json({ status: "ERROR", reason: "Invalid or used challenge" });

  try {
    const sigBuf  = Buffer.from(sig,  "hex");
    const keyBuf  = Buffer.from(key,  "hex");
    const msgBuf  = Buffer.from(k1,   "hex");
    const hash    = crypto.createHash("sha256").update(msgBuf).digest();
    const valid   = secp256k1.ecdsaVerify(sigBuf, hash, keyBuf);
    if (!valid)
      return res.json({ status: "ERROR", reason: "Invalid signature" });
  } catch (e) {
    return res.json({ status: "ERROR", reason: "Verification failed" });
  }

  // Mark used → store session → upsert profile
  db.prepare("UPDATE challenges SET used = 1 WHERE k1 = ?").run(k1);
  db.prepare("INSERT OR REPLACE INTO sessions (k1, lnkey, authed_at) VALUES (?, ?, ?)")
    .run(k1, key, Date.now());

  const existing = db.prepare("SELECT * FROM profiles WHERE lnkey = ?").get(key);
  if (!existing) {
    db.prepare("INSERT INTO profiles (lnkey, created_at, last_seen) VALUES (?, ?, ?)")
      .run(key, Date.now(), Date.now());
  } else {
    db.prepare("UPDATE profiles SET last_seen = ? WHERE lnkey = ?")
      .run(Date.now(), key);
  }

  res.json({ status: "OK" });
});

// ── ROUTE: poll for JWT ───────────────────────────────────────
app.get("/api/lnauth/verify", (req, res) => {
  const session = db
    .prepare("SELECT * FROM sessions WHERE k1 = ?")
    .get(req.query.k1);
  if (!session) return res.json({ status: "pending" });

  const token   = jwt.sign({ lnkey: session.lnkey }, JWT_SECRET,
                             { expiresIn: "7d" });
  const profile  = db.prepare("SELECT * FROM profiles WHERE lnkey = ?")
                      .get(session.lnkey);
  res.json({ status: "ok", token, profile });
});

// ── ROUTE: register wallet profile ───────────────────────────
app.post("/api/profile/register", (req, res) => {
  const auth = req.headers.authorization?.split(" ")[1];
  if (!auth) return res.status(401).json({ error: "Unauthorized" });

  let payload;
  try   { payload = jwt.verify(auth, JWT_SECRET); }
  catch { return res.status(401).json({ error: "Invalid token" }); }

  const { btc_address, alias, metadata } = req.body;
  db.prepare(`
    UPDATE profiles
    SET btc_address = ?, alias = ?, metadata = ?
    WHERE lnkey = ?
  `).run(
    btc_address  || null,
    alias        || null,
    JSON.stringify(metadata || {}),
    payload.lnkey
  );
  res.json({ status: "ok" });
});

// ── ROUTE: get own profile ────────────────────────────────────
app.get("/api/profile/me", (req, res) => {
  const auth = req.headers.authorization?.split(" ")[1];
  if (!auth) return res.status(401).json({ error: "Unauthorized" });
  let payload;
  try   { payload = jwt.verify(auth, JWT_SECRET); }
  catch { return res.status(401).json({ error: "Invalid token" }); }

  const profile = db.prepare(`
    SELECT lnkey, btc_address, alias, metadata, created_at
    FROM profiles WHERE lnkey = ?
  `).get(payload.lnkey);
  res.json(profile);
});

app.listen(PORT, () =>
  console.log(`[LNAUTH] Listening on :${PORT}`)
);
EOF

API Endpoint Reference

Endpoint Method Auth Description
/api/lnauth/init GET None Generate k1 challenge + LNURL + QR data URI
/api/lnauth/callback GET None (wallet) Receive sig+key from wallet, verify, store session
/api/lnauth/verify GET None Frontend polls for JWT once wallet has signed
/api/profile/register POST Bearer JWT Register BTC address, alias, metadata to ln profile
/api/profile/me GET Bearer JWT Return current user's profile record

Docker Deployment

YAML docker-compose.yml 
cat > docker-compose.yml << 'EOF'
version: "3.8"
services:
  lnauth:
    image: node:20-alpine
    working_dir: /app
    volumes:
      - ./:/app
    command: sh -c "npm install && node server.js"
    environment:
      - JWT_SECRET=${JWT_SECRET}
      - BASE_URL=${BASE_URL}
      - PORT=3000
    ports:
      - "3000:3000"
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://localhost:3000/api/lnauth/init"]
      interval: 30s
      timeout: 10s
      retries: 3
EOF
ENV .env 
cat > .env << 'EOF'
JWT_SECRET=$(openssl rand -hex 32)
BASE_URL=https://auth.yourdomain.com
PORT=3000
EOF
BASH Start service 
# Start

# View logs
docker compose logs -f lnauth

# Verify endpoint

Environment Variables

Variable Example Required Notes
JWT_SECRET openssl rand -hex 32 Required Never commit to source control. Rotate periodically.
BASE_URL https://auth.yourdomain.com Required Must be publicly reachable — wallets callback to this URL.
PORT 3000 Optional Defaults to 3000. Change if port is occupied.
⚠
BASE_URL Must Be Publicly Accessible Lightning wallets make the callback HTTP request from the user's mobile device or node. Your BASE_URL must be resolvable from the public internet. Use a reverse proxy (nginx, Caddy) with a valid TLS certificate. Self-signed certs will be rejected by most wallets.

JAVASCRIPT Page Settings → Before </body> 
<script>
const AUTH_SERVER = "https://auth.yourdomain.com";

async function initLNAuth() {
  const res                = await fetch(`${AUTH_SERVER}/api/lnauth/init`);
  const { k1, lnurl, qr } = await res.json();

  document.getElementById("lnauth-qr").src           = qr;
  document.getElementById("lnauth-lnurl").textContent = lnurl;
  document.getElementById("lnauth-link").href        = `lightning:${lnurl}`;

  // Poll for auth completion (2s interval)
  const poller = setInterval(async () => {
    const check = await fetch(`${AUTH_SERVER}/api/lnauth/verify?k1=${k1}`);
    const data  = await check.json();
    if (data.status === "ok") {
      clearInterval(poller);
      localStorage.setItem("sascosm_token",   data.token);
      localStorage.setItem("sascosm_profile", JSON.stringify(data.profile));
      onAuthSuccess(data.profile);
    }
  }, 2000);
}

function onAuthSuccess(profile) {
  document.getElementById("login-panel").style.display    = "none";
  document.getElementById("profile-panel").style.display  = "block";
  document.getElementById("wallet-key").textContent
    = profile.lnkey.substring(0, 16) + "...";

  if (profile.alias)
    document.getElementById("profile-alias").textContent
      = "Alias: " + profile.alias;
  if (profile.btc_address)
    document.getElementById("profile-btc").textContent
      = "BTC: " + profile.btc_address.substring(0, 14) + "...";

  // First login — show registration form
  if (!profile.btc_address && !profile.alias)
    document.getElementById("register-panel").style.display = "block";
}

async function registerWallet() {
  const token       = localStorage.getItem("sascosm_token");
  const btc_address = document.getElementById("btc-address-input").value;
  const alias       = document.getElementById("alias-input").value;

  await fetch(`${AUTH_SERVER}/api/profile/register`, {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${token}`,
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ btc_address, alias })
  });

  document.getElementById("register-panel").style.display = "none";
  alert("Wallet registered to your sovereign profile.");
}

// ── Auto-restore session on page load ─────────────────────────
const token = localStorage.getItem("sascosm_token");
if (token) {
  fetch(`${AUTH_SERVER}/api/profile/me`, {
    headers: { "Authorization": `Bearer ${token}` }
  })
  .then(r => r.json())
  .then(profile => {
    if (profile.lnkey) onAuthSuccess(profile);
    else initLNAuth(); // Token expired
  });
} else {
  initLNAuth();
}
</script>

Privacy Properties

✓
No Email Required Zero PII collected at any point in the auth flow.
✓
No Password Authentication is purely cryptographic. No credential to phish or leak.
✓
Domain-Scoped Key Wallet derives a unique keypair per domain. The same user is unlinkable across sites.
✓
Pseudonymous by Default Server stores only the Lightning public key. Alias and BTC address are opt-in.
✓
No IP Logging Required The protocol has no dependency on IP address for identity or verification.
✓
Non-Custodial No custody of keys, funds, or credentials at any layer.
✓
Tor Compatible If middleware is exposed via onion address, fully air-gapped from clearnet identity.
✓
No KYC Surface Zero regulatory identity requirements in the default implementation.

Data Held Server-Side

Field Type User Choice Notes
lnkey hex pubkey Mandatory (auth) Domain-scoped. Not linkable to other sites. Not a real identity.
btc_address string Opt-in User provides during registration. Can be omitted entirely.
alias string Opt-in User-chosen pseudonym. No real name collected.
metadata JSON blob Opt-in Arbitrary profile data. Application-defined.
created_at / last_seen timestamp System Unix ms. No timezone or geolocation inferred.

Optional Extensions

🔐
On-Chain Address Verification
Before binding a Bitcoin address to a Lightning profile, require the user to sign a message with the private key of the on-chain address. Standard Bitcoin message signing — verifiable server-side with bitcoinjs-message.
✊
BUMP Device Binding
If the user has a BUMP KeyRing, Chain, or Band, bind the passive NFC device ID to the same profile record keyed by lnkey. Allows NFC proximity payments to resolve to a known sovereign profile.
🗄
Webflow CMS Gating
Once sascosm_token is set in localStorage, use Webflow Logic or a thin JS guard to show/hide CMS Collection content. Gate premium content, member pages, or protocol documentation behind Lightning identity.
⚡
MYRA OS Node Identity
Route the LNURL-Auth callback through your LNbits instance on the MYRA OS Docker stack. Creates a unified lnkey identity across BTCPay, BUMP, the SMILE Ledger, and all SASCOSM service surfaces.
🧅
Tor Onion Exposure
Expose the middleware via an onion address. Wallets that route via Tor will complete authentication without any clearnet IP exposure. Combine with the BTCPay Tor service already running on your stack.
🌐
Redis Session Store
Replace SQLite with Redis for horizontally scalable session management. Set TTL on challenge keys (5 min) and session keys (7 days) to match JWT expiry. Use ioredis as a drop-in replacement for the better-sqlite3 calls.
